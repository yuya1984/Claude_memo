# KQL(Kusto Query Language)基礎

Application Insights に蓄積されたログを検索・集計するための専用クエリ言語が KQL(Kusto Query Language)です。

## KQL とは

### 語源:ジャック=イヴ・クストーへのオマージュ

「Kusto」という名前は、フランスの海洋探検家・映画監督である**ジャック=イヴ・クストー(Jacques Cousteau)** にちなんで名付けられました。「未知の深海を探検する」というクストーのイメージが、「膨大なログデータの海を探索する(explore an ocean of data)」というこの製品のコンセプトと重なることから、この名前が採用されました。

### 開発の背景

このプロジェクトは、2014年に Microsoft のイスラエル研究開発拠点で、草の根的な取り組みとして始まりました。当初の目的は、Azure の各種サービスにおける高速でスケーラブルなログ・テレメトリ分析のニーズに応えることでした。その後2016年に Application Insights Analytics のバックエンドとして採用され、2018年の Microsoft Ignite で正式に「Azure Data Explorer」としてパブリックプレビューが発表されました。

KQL は、Azure Monitor・Application Insights・Microsoft Sentinel など、Microsoft のログ分析系サービス全般で使われる、**ログデータを検索・集計・可視化するための専用クエリ言語** です。

## SQL との違い

SQL に近い感覚で扱えますが、書き方の思想が少し異なります。

- SQL は「`SELECT 列 FROM テーブル WHERE 条件`」のように、先頭から条件を書く
- KQL は「`テーブル名` から始めて、`|`(パイプ)で処理を繋いでいく」という書き方をする

イメージとしては、テーブルというデータの塊に対して、パイプでフィルタ・整形・並び替えの処理を1段ずつ通していく、という考え方です。

```kusto
customEvents                  ← まずテーブルを指定
| where timestamp > ago(1d)   ← パイプでフィルタを1段追加
| project timestamp, name     ← 見る列を絞る
| order by timestamp desc     ← 並び替え
```

## クエリ入力画面への行き方

1. **Azure Portal**(`https://portal.azure.com`)にサインイン
2. 上部の検索バーで、連携している Application Insights リソース名を検索し、開く
3. 左側メニューの「**監視(Monitoring)**」セクションから「**ログ(Logs)**」を選択
4. 初回アクセス時、サンプルクエリを提案するポップアップが出ることがある。右上や画面内の「×」で閉じてよい
5. 画面中央に**クエリエディタ**(白い入力欄)が表示される。ここに直接 KQL を入力する
6. 入力後、エディタ上部の「**実行(Run)**」ボタンをクリック(もしくは `Shift + Enter`)
7. エディタの下側に結果がテーブル形式で表示される

## 主要な演算子

| 演算子 | 意味 |
|---|---|
| `where` | 条件で行を絞り込む |
| `extend` | 新しい列を作る(既存の値を加工・変換して追加) |
| `project` | 表示する列を指定する |
| `order by` | 並び替える |
| `take N` | 先頭 N 件だけ表示する |
| `tostring(...)` | 動的型の値を文字列型に変換する |
| `isnotempty(...)` | 値が空でないかを判定する |
| `ago(1d)` | 現在時刻から遡った期間を指定する(`1d` = 1日、`7d` = 7日など) |

`customDimensions` のようにログが JSON 形式で拡張データを持っている場合、`customDimensions['text']` のようにキー名を指定して値を取り出せます。多くの場合、動的型のままだと扱いにくいため `tostring()` で文字列化します。
