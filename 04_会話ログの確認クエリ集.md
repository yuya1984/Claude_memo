# 会話ログの確認クエリ集

[02_ApplicationInsights連携セットアップ.md](02_ApplicationInsights連携セットアップ.md) の設定が完了していることを前提に、Copilot Studio の会話ログを Application Insights 上で確認するための実践クエリをまとめます。KQL の基本文法は [03_KQL基礎.md](03_KQL基礎.md) を参照してください。

## 結論:会話ログはどのテーブルに入っているか

Teams などでユーザーとエージェントが交わした会話のログ(発言内容、送受信のタイミング、トリガーされたトピックなど)は、主に **`customEvents`** テーブルに記録されます。

ユーザーの具体的な発言内容やエージェントからの返答テキストは、`customEvents` テーブル内の **`customDimensions`** という列に JSON 形式の拡張データとして内包されています。

用途別に3つのテーブルを使い分けます。

| テーブル | 内容 |
|---|---|
| `customEvents` | ユーザーからのメッセージ、ボットからの応答、セッションの開始・終了などの主要な会話イベント |
| `traces` | エラーや AI からの詳細な診断メッセージ |
| `dependencies` | エージェントが裏側で Power Automate のフローなどを呼び出した際の処理時間や結果 |

## クエリ手順

1. Application Insights リソースを開き、左側メニュー「監視」セクションの「ログ(Logs)」をクリック
2. 画面中央のクエリ入力エリアに、以下のクエリをコピー&貼り付け
3. 右上の「実行(Run)」ボタンをクリック
4. 画面下部の「結果」タブで一覧を確認。各行の左端の `>`(展開)アイコンをクリックすると `customDimensions` の全データが展開され、ユーザー ID やトリガーされたトピック名などの詳細情報も確認できる

## サンプルクエリ

### ① まずはテーブルの中身をそのまま覗いてみる(最もシンプル)

```kusto
customEvents
| take 50
```

`take 50` は「先頭50件だけ表示する」という意味です。まずはどんな列があるかを確認する目的で使います。

### ② 会話テキストを取り出す(実務で使う本命クエリ)

```kusto
customEvents
| where timestamp > ago(1d)
| extend messageText = tostring(customDimensions['text'])
| extend userId = tostring(customDimensions['fromId'])
| extend channel = tostring(customDimensions['channelId'])
| where isnotempty(messageText)
| project timestamp, userId, channel, messageText
| order by timestamp desc
```

**各行の意味:**

- `where timestamp > ago(1d)`:直近24時間分に絞る(`ago(7d)` にすれば7日前まで)
- `extend`:新しい列を作る。`customDimensions` は JSON 形式で入っているため、`['text']` のようにキー名を指定して中の値を取り出す
- `tostring(...)`:取り出した値を文字列型に変換(そのままだと動的型のままで扱いにくいため)
- `where isnotempty(messageText)`:テキストが空でない行だけに絞る(システムイベントなどノイズを除外)
- `project`:最終的に表示する列を指定
- `order by timestamp desc`:新しい順に並び替え

### ③ Teams チャネルのメッセージ送受信イベントに絞る

```kusto
customEvents
| where name in ("BotMessageReceived", "BotMessageSend")
| order by timestamp desc
| project timestamp, eventName = name, messageText = customDimensions.text, channel = customDimensions.channelId
```

- `messageText`:ユーザーが入力した文字(例:「おはよう」)や、エージェントが返したテキストが表示される
- `eventName`:`BotMessageReceived`(ユーザーからの受信)か `BotMessageSend`(エージェントからの送信)かが判別できる
- `channel`:会話が行われたチャネル(Teams の場合は `msteams`)

### ④ テスト会話を除外して、本番の会話だけに絞る

```kusto
customEvents
| extend isDesignMode = tostring(customDimensions['designMode'])
| where isDesignMode == "False"
| extend messageText = tostring(customDimensions['text'])
| where isnotempty(messageText)
| project timestamp, messageText
| order by timestamp desc
```

### ⑤ エラーや診断メッセージを見たい場合(`traces` テーブル)

```kusto
traces
| where timestamp > ago(1d)
| where severityLevel >= 2
| project timestamp, message, severityLevel
| order by timestamp desc
```

`severityLevel >= 2` は「警告(Warning)以上」の重要度に絞る条件です。

### ⑥ Power Automate フローなど外部呼び出しの処理時間を見たい場合(`dependencies` テーブル)

```kusto
dependencies
| where timestamp > ago(1d)
| project timestamp, name, target, duration, success
| order by timestamp desc
```

`duration` は処理にかかった時間(ミリ秒)、`success` は成功・失敗のフラグです。

### ⑦ 特定のユーザーの会話だけを追いたい場合

```kusto
customEvents
| extend messageText = tostring(customDimensions['text'])
| extend userId = tostring(customDimensions['fromId'])
| where userId == "対象のユーザーID"
| where isnotempty(messageText)
| project timestamp, messageText
| order by timestamp desc
```

## 進め方のヒント

まずは①の `take 50` クエリを実行して、実際に `customDimensions` の中に `text` フィールドが入っているかを自分の環境で確認するのが確実な第一歩です。列名が想定と違う場合(例えば `text` ではなく別の名前で入っている場合)は、実際の列構成に合わせてクエリを調整してください。

## 代替手段:公式ダッシュボードの利用

クエリを毎回手動で実行する代わりに、Microsoft が公式に提供している専用の可視化ダッシュボードを使うこともできます。

1. Application Insights リソースの左側メニュー「監視」セクションの「ブック」をクリック
2. ギャラリー一覧から「Copilot Studio Dashboard」を開く
3. クエリを書くことなく、会話の総数・エラー率・使用されたトピックなどの分析データをグラフや表で確認できる
